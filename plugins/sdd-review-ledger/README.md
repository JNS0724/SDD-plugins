# sdd-review-ledger

vibe coding 时，**自动化辅助评审** SDD 文档↔文档、代码↔文档之间的偏差。

与上一代 `sdd-drift-check` 的根本区别：本项目**不让工具去"算偏差"**（那是语义判断，无确定性算法），而是把工具重新定位为「**评审编排器 + 评审账本**」——可靠地在变更后把材料端到回路里唯一够格的语义裁判（agent 本身）面前请它评审，并用内容哈希记住裁决。偏差判断永远由 LLM 做。

## 一分钟理解

- **工具保证**（机械）：内容哈希追踪变更；编辑时在工具结果里端出"该评审的材料"；落人可见 `.sdd-review-todo.md` 兜底；跨会话、子代理透明、永不阻断。
- **工具委托**（语义）：是否真漂移 / 文档要不要改 / 归属哪个 change-dir → 全交 agent。
- **工具坦白做不到**：不保证"代码真符合 design"——评审质量 = LLM 质量。它只保证"评审被可靠端到裁判面前并留痕"。

设计血脉与被推翻的旧方案见架构文档 §11。
